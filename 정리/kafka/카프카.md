
 Kafka
 ======



카프카
- 낮은 대기 시간
- 높은 처리량으로 데이터 처리가 용이
- 빠르다 (디스크 캐싱 및 메모리 매핑 파일에 의존하기 때문)
- 탄력적이다
- 메모리 대신 파일 시스템에 의존
- 모든 데이터는 디스크에 저장 -> 복제본을 유지하여 데이터 손실 방지
- 대부분의 운영체제는 메모리 영역이 커널의 페이지 캐시에 있음 -> 사용자 메모리에 복사본 생성 X -> 캐시가 더 빠름
- kafka 내부엔 파티션이 존재 -> 확장 기능 (scale out)


토픽 
- 불변(Immutable)의 추가 전용 구조에서 데이터(로그, 이벤트) 추가
- 불변성으로 인해 데이터의 모든 기록을 보유 및 GET 가능(어느 시점이나)
